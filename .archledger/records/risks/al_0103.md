The storage metadata file tracks next-number counters for each record type prefix. If the metadata becomes stale (e.g., after manual file operations or rapid concurrent creation), new records may receive colliding IDs or filenames. The `--repair-counters` flag on `check` recomputes counters from the actual files on disk.
