The storage metadata file tracks the next ledger number. If metadata becomes stale
or numbered fragments are manually deleted, new records can collide with existing
history. `archledger doctor --repair` recomputes `storage.yaml.next_number` and
creates archive tombstones for missing non-section IDs so numbering stays gapless
without renumbering existing records.
